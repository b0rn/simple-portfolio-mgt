# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please send a detailed report to the project maintainer via:

- **Email:** [contact@vleveneur.com](mailto:contact@vleveneur.com)

### What to include

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if any)

### Response timeline

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 7 days
- **Fix or mitigation:** Dependent on severity

## Security Best Practices

This project follows these security practices:

- Dependencies are regularly updated and audited
- Secrets are never committed to the repository
- Authentication and authorization are enforced on all API endpoints
- Input validation and sanitization are applied at system boundaries
- Container images are scanned for known vulnerabilities
- SonarQube is used for static code analysis

## Disclosure Policy

Once a vulnerability is confirmed and a fix is available, a security advisory will be published with appropriate credit to the reporter.
